# 03 Data Dictionary - 企业画像字段字典

## 目的
统一企业画像字段定义、数据类型、校验规则和必填约束，确保前后端与测试对数据理解一致。

## 适用范围
适用于企业端信息采集、匹配计算与工单关联涉及的企业基础数据。

## 定义与术语
- 必填：前端提交时必须提供。
- 选填：可为空，不阻断初筛。
- 枚举：取值必须在预定义集合内。
- 正则校验：格式校验规则。

## 主内容
### 1. 字段字典
| 字段名 | 类型 | 必填 | 示例 | 校验规则 | 说明 |
|---|---|---|---|---|---|
| enterprise_name | string | 是 | 上海某科技有限公司 | 长度 2-120 | 企业名称 |
| uscc | string | 是 | 9131XXXXXXXXXXXXXX | 18位统一社会信用代码格式 | 统一社会信用代码 |
| region_code | string | 是 | SH-PD | 枚举：上海各区代码 | 企业所在区 |
| industry_code | string | 是 | C39 | 枚举：行业分类码 | 企业行业 |
| contact_name | string | 是 | 张三 | 长度 2-30 | 联系人姓名 |
| contact_mobile | string | 是 | 13800138000 | `^1[3-9]\d{9}$` | 联系人手机号 |
| employee_scale | string | 否 | 50-99 | 枚举区间 | 员工规模 |
| revenue_range | string | 否 | 1000万-5000万 | 枚举区间 | 年营收区间 |
| rd_ratio | number | 否 | 6.5 | 0-100 | 研发投入占比（%） |
| qualification_tags | string[] | 否 | ["高新技术企业"] | 标签枚举 | 资质证书 |
| ip_count | integer | 否 | 12 | >=0 | 知识产权数量 |

### 2. 枚举建议
1. `region_code`：SH-HQ、SH-PD、SH-MH、SH-XH 等。
2. `employee_scale`：1-9、10-49、50-99、100-299、300+。
3. `revenue_range`：0-100万、100万-1000万、1000万-5000万、5000万+。

### 3. 数据质量规则
1. 必填字段缺失时不得提交匹配任务。
2. 选填字段缺失时允许提交，但在结果页提示可提升精度项。
3. 同一 `uscc` 在同一租户下视为同一企业档案。

### 4. 安全与隐私
1. 联系方式展示需脱敏。
2. 存储与日志不得明文暴露验证码。
3. 导出数据遵循最小化原则。

## 验收标准
1. 前后端按字段字典实现后无字段歧义。
2. 关键字段校验规则可被自动化测试覆盖。
3. 必填/选填行为与 PRD 一致。
4. 匹配引擎所需字段在本字典均有定义。
