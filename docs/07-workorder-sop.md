# 07 Workorder SOP - 转人工工单规范

## 目的
统一人工工单处理流程、状态机、SLA 和回呼规则，确保服务闭环可跟踪、可复盘。

## 适用范围
适用于企业端发起的政策咨询类工单。

## 定义与术语
- 工单状态：工单生命周期阶段。
- SLA：服务时效承诺。
- 首响：首次人工联系用户。

## 主内容
### 1. 工单状态机
1. `pending`（待受理）
2. `processing`（处理中）
3. `resolved`（已完成）
4. `closed`（已关闭，用户确认或超时关闭）

### 2. 状态流转规则
1. 创建工单后进入 `pending`。
2. 人工接单后更新为 `processing`。
3. 给出结论并同步用户后更新为 `resolved`。
4. 用户确认或超时后更新为 `closed`。

### 3. SLA 建议
1. 首响时长：工作时段内 4 小时内。
2. 处理时长：标准咨询 2 个工作日内给出结论。
3. 超时工单：自动标红并提醒。

### 4. 回呼规则
1. 优先按用户预约时间联系。
2. 首次未接通，至少补呼 2 次。
3. 回呼记录必须写入工单日志。

### 5. 记录规范
1. 必填：问题分类、处理结论、建议动作。
2. 附件：用户上传材料与顾问补充文件。
3. 敏感信息仅授权角色可见。

## 验收标准
1. 状态机流转无非法跳转。
2. SLA 可被统计，超时可被追踪。
3. 回呼动作全量留痕。
4. 工单关闭后可完整复盘处理过程。
