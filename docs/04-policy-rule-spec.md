# 04 Policy Rule Spec - 政策规则规范

## 目的
规范政策结构化、匹配规则模板和解释文本生成规则，确保匹配结果可计算、可解释、可追溯。

## 适用范围
适用于政策数据入库、规则维护、匹配引擎输出与结果解释模块。

## 定义与术语
- 硬条件：不满足即不可申报。
- 评分条件：用于排序和优先级判断。
- 缺口项：当前不满足但可补齐的信息或资质。
- 解释文本：匹配结果中给用户可读的原因说明。

## 主内容
### 1. 政策结构模板
```json
{
  "policy_id": "SH-2026-0001",
  "title": "示例政策",
  "region_code": "SH-PD",
  "level": "city|district",
  "effective_from": "2026-01-01",
  "effective_to": "2026-12-31",
  "source_url": "https://example.gov.cn/policy/1",
  "hard_conditions": [],
  "scoring_conditions": [],
  "required_materials": []
}
```

### 2. 匹配规则模板
1. 先执行硬条件过滤，命中全部硬条件标记为 `eligible`。
2. 未命中硬条件标记为 `potential`，并按缺口数量执行分值扣减。
3. 输出两类结果：`eligible`、`potential`。
4. 每条结果必须附 `reasons` 与 `missing_items`。

### 3. 解释文本规范
1. 文本必须包含“命中依据”或“未命中依据”。
2. 引用至少一个政策来源（链接或条款定位）。
3. 禁止使用无法核实的主观判断词（如“大概率一定能过”）。

### 4. 政策变更同步规则
1. 政策失效/更新后必须更新 `effective_to` 或版本号。
2. 变更后触发重匹配任务。
3. 相关接口字段变更需同步 `docs/05-api-contract-v1.md`。

### 5. 示例规则
- 规则：`rd_ratio >= 5` 且 `qualification_tags` 包含“高新技术企业”。
- 命中解释：研发投入占比满足要求，且具备高新技术企业资质。
- 缺口解释：尚未识别到高新技术企业资质，建议补充证明材料后再评估。

## 验收标准
1. 任一政策可按模板完整结构化。
2. 匹配输出均具备分类、原因、缺口和来源。
3. 政策更新后可在 1 个版本周期内完成规则同步。
4. 测试可根据规范构造可重复用例。
