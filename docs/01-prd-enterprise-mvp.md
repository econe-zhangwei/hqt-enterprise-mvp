# 01 PRD - 企业端 MVP

## 目的
定义企业端 MVP 的功能需求、页面行为、交互约束与验收标准，为设计、开发、测试提供统一输入。

## 适用范围
仅适用于企业端 Web H5 MVP。
不包含园区端界面与在线申报系统。

## 定义与术语
- 摘要结果：用于快速查看数量与分布的简化匹配结论。
- 完整结果：登录后可查看的完整政策列表与解释信息。
- 工单：用户发起人工协助请求后的服务单据。
- 政策引用：回答中明确标注的政策来源链接或条款标识。

## 主内容
### 1. 用户故事
1. 作为企业用户，我希望输入公司信息后快速知道有哪些政策可申报。
2. 作为企业用户，我希望看到“为什么能报/不能报”，而不是只给结果。
3. 作为企业用户，我希望有疑问时先问 AI，再决定是否转人工。
4. 作为企业用户，我希望提交人工协助后可查看处理进度。

### 2. 页面清单
1. 登录页（默认账号密码）
2. 工作台总览页
3. 企业信息填写页
4. 匹配结果页
5. 政策详情页
6. AI 问答页
7. 转人工工单页

### 3. 页面功能需求
#### 3.1 登录与总览
1. 用户先登录，再进入工作台。
2. 总览展示匹配统计、工单状态、下一步建议。
3. 总览提供到企业信息、匹配、AI、工单的导航入口。

#### 3.2 企业信息填写页
1. 必填字段：企业名称、统一社会信用代码、所在区、行业、联系人手机号。
2. 选填字段：员工规模、营收区间、研发投入、资质证书、知识产权数量。
3. 提交后触发匹配任务。

#### 3.3 匹配结果页
1. 分组展示：可申报、需完善。
2. 每条政策显示：政策标题、支持方式、命中理由、缺口项、下一步建议。
3. 登录后展示完整结果；未登录不可访问业务页面。

#### 3.4 政策详情页
1. 展示政策名称、发布部门、生效时间、适用对象、支持方式、条件明细。
2. 提供原文链接与更新时间。
3. 提供“转人工协助”入口。

#### 3.5 AI 问答页
1. 支持针对当前企业画像的问答。
2. 回答必须包含引用来源。
3. 无法确定时触发“建议转人工”。

#### 3.6 转人工工单页
1. 提交项：问题类型、补充说明、附件、预约联系时间。
2. 状态：待受理、处理中、已完成。
3. 可查看处理记录与最近更新时间。

### 4. 状态与交互规则
1. 登录优先
- 阶段一：用户先使用账号密码登录（开发默认账号）。
- 阶段二：登录后进入企业工作台，执行企业画像、匹配、问答、工单流程。
2. 匹配中状态
- 提交后显示“匹配处理中”，支持轮询获取结果。
3. 空结果状态
- 当可申报政策为 0 时，展示需完善建议和转人工入口。

### 5. 非功能要求
1. 页面首屏可用时间满足移动端基本体验。
2. 敏感信息脱敏展示（如手机号中间四位）。
3. 页面行为需可埋点（提交、登录、查看、转人工）。

### 6. 评审机制
1. 需求评审：本文件 + `docs/02-user-flow-and-ia.md`。
2. 技术评审：本文件关联 `docs/03/04/05`。

## 验收标准
1. 7 个页面需求与状态均可被开发直接实现。
2. 登录优先行为在文档中无冲突。
3. 每个核心动作均有明确触发条件与结果反馈。
4. 测试可依据本文件独立编写用例。
