# 08 Release Checklist - 发布前强制校对清单

## 目的
在发布前执行统一校对，确保文档、接口、字段和实现保持一致，降低上线偏差风险。

## 适用范围
适用于企业端 MVP 每次发布前的文档与实现一致性检查。

## 定义与术语
- 强制校对：发布前必须完成，不可跳过。
- 一致性：文档定义与实际实现对齐。
- 阻塞项：未通过则禁止发布。

## 主内容
### 1. 文档一致性
- [ ] `00` 范围边界与当前版本目标一致。
- [ ] `01` 页面与交互已覆盖本次发布内容。
- [ ] `02` 流程变更已更新。
- [ ] `09` 已记录本次文档改动。

### 2. 字段与规则一致性
- [ ] `03` 字段字典与前端表单字段一致。
- [ ] `04` 匹配规则与后端实现一致。
- [ ] 新增/废弃字段均有兼容说明。

### 3. 接口一致性（SSOT）
- [ ] `05` 接口契约与网关路由一致。
- [ ] 请求/响应字段与前端解析一致。
- [ ] 错误码与异常提示一致。

### 4. AI 与工单一致性
- [ ] `06` 引用规范与拒答策略已上线。
- [ ] `07` 工单状态机与实际流转一致。
- [ ] 转人工触发条件可复现。

### 5. 发布演练（必做）
- [ ] 样本企业 A（高匹配）走通全流程。
- [ ] 样本企业 B（低匹配）走通 AI->转人工。
- [ ] 样本企业 C（信息缺失）验证补录提示。
- [ ] 1 条政策变更触发重匹配验证通过。

### 6. 发布结论
- [ ] 可发布
- [ ] 不可发布（说明阻塞项）

## 验收标准
1. 清单项全部执行并有记录。
2. 存在阻塞项时自动终止发布。
3. 发布后问题可追溯到具体清单项。
4. 评审人员可独立复核。
