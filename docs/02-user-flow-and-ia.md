# 02 User Flow and IA - 企业端 MVP

## 目的
定义企业端信息架构（IA）与关键业务流程，保证产品链路清晰、可实现、可验证。

## 适用范围
仅适用于企业端 Web H5 的用户流程与页面结构设计。

## 定义与术语
- IA：信息架构，页面与功能入口的组织方式。
- 主流程：用户完成核心目标的最短路径。
- 分流点：用户可选择继续自助或转人工的节点。

## 主内容
### 1. IA 结构
1. 登录页
2. 企业工作台总览
3. 企业信息填写
4. 政策匹配结果
5. 政策详情
6. AI 问答
7. 转人工工单

### 2. 关键主流程
#### 流程 A：登录后完成匹配
1. 用户先完成账号密码登录。
2. 进入工作台并填写企业信息。
3. 提交后触发匹配任务。
4. 返回完整政策结果并可查看详情。

#### 流程 B：基于结果追问
1. 用户在匹配结果中进入政策详情。
2. 点击“问 AI”进入问答页。
3. AI 返回解释与引用。
4. 若 AI 不确定，推荐用户转人工。

#### 流程 C：转人工
1. 用户从结果页/详情页/问答页进入工单页。
2. 填写诉求并提交。
3. 查看工单状态更新。

### 3. 分流与兜底策略
1. 自助优先：先看结果、先问 AI。
2. 人工兜底：复杂场景统一走工单。
3. 不支持自动申报跳转，避免错误承诺。

### 4. 页面跳转矩阵
| 来源页面 | 目标页面 | 触发条件 |
|---|---|---|
| 登录页 | 工作台总览 | 登录成功 |
| 工作台总览 | 企业信息填写 | 点击企业信息导航 |
| 企业信息填写 | 匹配结果 | 提交并匹配成功 |
| 匹配结果 | 政策详情 | 点击政策卡片 |
| 匹配结果/政策详情 | AI问答 | 点击问AI |
| 匹配结果/政策详情/AI问答 | 工单页 | 点击转人工 |

### 5. 失败与异常流程
1. 匹配失败：提示重试并保留用户填写内容。
2. 登录失败：保留当前上下文，允许再次验证。
3. 工单提交失败：提示失败原因并支持再次提交。

## 验收标准
1. IA 中每个页面均有明确上下游关系。
2. 主流程 A/B/C 无闭环缺失。
3. 异常流程可覆盖失败重试场景。
4. 前端和测试可直接按流程图实现与验证。
